# 工作流引擎

ShadowBoard 通过 DAG（有向无环图）编排任务步骤，支持跨步骤上下文接力与模板化执行。

---

## 🔄 概念说明

工作流由多个步骤 (Step) 组成，当前支持：
*   **任务接力 (Context Relay)**：使用 `{prev_result}` 引用前序环节的输出。
*   **顺序执行**：严格按照依赖关系运行。
*   **并行步骤**：支持并行步骤定义（由步骤类型控制）。
*   **条件/延时步骤**：支持条件与延时类型步骤。

## 🎭 核心角色

| 角色 | 职能描述 |
| :--- | :--- |
| **CMO (市场)** | 负责市场规模、竞争对手与用户痛点分析。 |
| **CTO (技术)** | 负责技术可行性、架构复杂度与技术债评估。 |
| **CFO (财务)** | 负责成本收益、盈利模式与资金风险审计。 |
| **Chairman (董事长)** | 负责最终意见汇总与行动指令下达。 |
