# 核心服务

ShadowBoard 当前是单进程应用中的“服务层模块化架构”，并非远程微服务部署。核心服务如下：

---

## 📊 任务追踪 (TaskTracker)
负责全生命周期的任务管理：
*   **状态机控制**：Pending -> Running -> Completed/Failed -> Retrying。
*   **持久化**：所有任务状态与事件日志均存储在 SQLite (`tasks.db`)。
*   **依赖解析**：自动检查前置任务是否成功完成。

## 💾 记忆存储 (MemoryStore)
管理董事会的对话历史：
*   **多会话隔离**：支持并行开启多个不同的议案。
*   **上下文窗口**：智能控制传递给 Web AI 的历史消息数量。
*   **反向检索**：O(1) 效率的历史记录查询。

## 📈 监控报表 (Monitor)
提供系统运行状况的实时视图：
*   **性能指标**：记录每个专家的响应时长与成功率。
*   **健康检查**：自动检测浏览器引擎、数据库与网络链路。
*   **告警管理**：当任务连续失败或延迟过高时触发告警。
